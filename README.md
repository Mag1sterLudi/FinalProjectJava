MyShortLink — Сервис коротких ссылок

Итоговый проект по курсу Java-basic. Это консольное приложение, которое превращает длинные и неудобные ссылки в короткие коды.

### Как запустить проект

Открой проект в IntelliJ IDEA как Maven-проект.
Проверь, что в папке с проектом есть файл application.properties.
Запусти класс shortlink.App (зеленая стрелочка).
Следуй подсказкам в консоли. Для создания ссылки обязательно вводи её с http:// или https://.

Основные функции (что я реализовал):

Уникальность: Даже если два человека сократят одну и ту же ссылку (например, Google), они получат разные короткие коды.
Лимиты переходов: Можно ограничить количество кликов. Например, поставить 5 — и на 6-й раз ссылка уже не откроется.
Срок жизни (TTL): Каждая ссылка имеет время жизни. Если время вышло, программа её удаляет и не дает перейти.
Идентификация: Система узнает пользователя по его UUID (уникальному ID). Можно выйти и зайти под своим старым ID, чтобы снова управлять своими ссылками.
Безопасность: Программа проверяет владельца. Ты не можешь удалить ссылку, которую создал другой пользователь.
Автоматизация: При переходе по ссылке программа сама пытается открыть браузер.

### Описание классов:

### Пакет model (Данные):

**User**: Хранит ID пользователя. Я переопределил в нем equals и hashCode, чтобы программа понимала: два объекта с одинаковым ID — это один и тот же человек.

**Link**: Тут лежат все данные о ссылке: длинный URL, короткий код, лимит кликов, счетчик переходов и время, когда ссылка «протухнет».

### Пакет service (Логика):

**LinkService**: Это методы для создания ссылки (генерация короткого кода), проверки лимитов и времени жизни. Тут же происходит поиск ссылок конкретного юзера и проверка прав доступа перед удалением.

### Пакет util (Инструменты):

**Config**: Класс, который лезет в файл application.properties и достает оттуда настройки (время жизни ссылок и лимиты).

### Главный класс:
**App**: Тут лежит наше консольное меню (switch-case). Он отвечает взаимодействие с пользователем, ввод данных и вывод сообщений об ошибках.

### Настройки (application.properties)

### Чтобы не менять код каждый раз, важные параметры вынесены в отдельный файл.

link.ttl.hours=24 — сколько часов будет работать ссылка с момента создания.
link.default.limit=10 — сколько переходов будет по умолчанию, если не ввели число.

### Тестирование

В проекте написано 15 Unit-тестов (JUnit 5). Я постарался проверить все важные моменты:

Блокируется ли ссылка после исчерпания лимита.
Выдается ли ошибка, если ссылка «протухла» по времени.
Работает ли защита от удаления чужих ссылок.
Правильно ли находятся ссылки при смене пользователя.

Запуск тестов:
Нажми правой кнопкой на папку src/test/java и выбери Run 'All Tests'. Или через терминал: mvn test.
